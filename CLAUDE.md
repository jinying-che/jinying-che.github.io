# Project

Hugo blog (jinying-che.github.io). Content in `content/posts/`.

## Tech Stack
- Hugo static site generator
- Markdown for content
- Custom CSS in `static/style.css`

## Preferences
- LeetCode solutions: use golang (focus on algorithm, not language)
- Explain algorithms step-by-step with ASCII diagrams
- Keep code clean — avoid syntax sugar unless asked
- Use tables for comparisons
- Dry run with concrete examples
- No auto-commit — only commit when asked
- Always show proposed changes in the CLI first and wait for user confirmation before editing post files
